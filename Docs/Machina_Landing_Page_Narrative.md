## Section 1 — What Machina is

### Title
**More than a chatbot. More than an automation script.**

### Body
Machina is an AI-native operating layer for technical work.

Instead of only answering questions, it helps users turn intent into visible, governed execution inside a workspace. That means planning, tool use, approvals, event traces, runtime metrics, and reusable workflows all live in the same shell.

Machina is built for environments where trust, control, and transparency matter as much as raw intelligence.

## Section 3 — How it works

### Title
**How Machina works**

### Step 1
**Understand intent**  
Machina turns a goal into a structured task.

### Step 2
**Build a governed plan**  
It selects tools, orders steps, and marks risky actions for approval.

### Step 3
**Execute and show its work**  
Machina runs the workflow while exposing task state, events, tool results, and metrics.

## Section 4 — Why it feels different

### Title
**Why Machina feels different**

### Body
Many AI systems can suggest.

Machina is built to **plan, execute, explain, and govern**.

That means:
- explicit tool boundaries
- visible task and step states
- human-in-the-loop approvals
- durable runtime history
- workflow and agent orchestration
- a workspace shell instead of a black box chat window

The result is an AI system that feels less like vague assistance and more like a controllable operating layer.
